# 네이버 웹마스터 도구 메타 태그 문제 해결

## 문제
네이버 웹마스터 도구에서 메타 태그를 찾을 수 없거나 로봇이 접근하지 못함

## 원인
1. Streamlit은 `st.markdown`으로 추가한 메타 태그가 `<head>` 섹션에 들어가지 않음
2. 네이버 봇이 JavaScript를 실행하지 않을 수 있음

## 해결 방법

### 방법 1: JavaScript로 메타 태그 동적 삽입 (적용 완료)
- JavaScript를 사용하여 `<head>` 섹션에 메타 태그를 동적으로 추가
- 배포 후 확인 필요

### 방법 2: HTML 파일 업로드 방식 (권장)
네이버 웹마스터 도구에서 제공하는 HTML 파일을 업로드하는 방법이 더 확실합니다.

**단계:**
1. 네이버 웹마스터 도구에서 "HTML 파일 업로드" 방식 선택
2. 제공된 HTML 파일 다운로드 (예: `naver_verification.html`)
3. 파일을 프로젝트 루트에 저장
4. Streamlit에서 해당 경로로 접근 가능하도록 설정

### 방법 3: robots.txt 확인
네이버 봇이 접근할 수 있는지 확인:
- `User-agent: *` 및 `Allow: /` 설정 확인
- 네이버 봇이 차단되지 않았는지 확인

## 확인 방법

### 1. 브라우저에서 확인
1. `https://unsedam.kr` 접속
2. 개발자 도구 열기 (F12)
3. Console 탭에서 JavaScript 실행 확인
4. Elements 탭에서 `<head>` 섹션에 메타 태그 확인

### 2. 페이지 소스 확인
1. 페이지에서 우클릭 → "페이지 소스 보기"
2. `<head>` 섹션에서 다음 메타 태그 확인:
   ```html
   <meta name="naver-site-verification" content="43dc017823a9c46420c367e23e62c5a3f0e0d99c" />
   ```

### 3. 네이버 봇 테스트
네이버 웹마스터 도구에서 "소유권 확인" 버튼을 다시 클릭하여 확인

## 추가 조치

### robots.txt 확인
현재 robots.txt 설정:
```
User-agent: *
Allow: /
Disallow: /static/
```

네이버 봇이 접근할 수 있도록 설정되어 있습니다.

### HTML 파일 업로드 방식 (대안)
JavaScript 방식이 작동하지 않으면 HTML 파일 업로드 방식을 사용하세요:
1. 네이버 웹마스터 도구에서 HTML 파일 다운로드
2. 파일명과 내용을 알려주시면 설정하겠습니다

## 참고사항
- 네이버 봇은 일반적으로 JavaScript를 실행하지 않습니다
- HTML 파일 업로드 방식이 가장 확실한 방법입니다
- 배포 후 몇 분에서 몇 시간이 걸릴 수 있습니다
