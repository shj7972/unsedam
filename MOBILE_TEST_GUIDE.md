# 모바일 테스트 가이드

## 모바일 디바이스에서 앱 확인하기

### 방법 1: 브라우저 개발자 도구 사용 (가장 간단)

1. **Chrome/Edge 개발자 도구 사용:**
   - `F12` 또는 `Ctrl+Shift+I` (Windows) / `Cmd+Option+I` (Mac)로 개발자 도구 열기
   - `Ctrl+Shift+M` (Windows) / `Cmd+Shift+M` (Mac)으로 모바일 뷰 전환
   - 상단의 디바이스 선택 메뉴에서 원하는 디바이스 선택 (iPhone, Galaxy 등)
   - 또는 크기를 직접 지정 (예: 375x667 for iPhone SE)

2. **테스트할 주요 해상도:**
   - iPhone SE: 375 x 667
   - iPhone 12/13: 390 x 844
   - Galaxy S20: 360 x 800
   - iPad: 768 x 1024

### 방법 2: 실제 모바일 디바이스에서 접근

1. **로컬 네트워크에서 접근:**
   ```bash
   # Streamlit 앱 실행 (네트워크 접근 허용)
   streamlit run app.py --server.address 0.0.0.0
   ```
   
2. **IP 주소 확인 (Windows):**
   ```bash
   ipconfig
   ```
   IPv4 주소를 확인하세요 (예: 192.168.0.100)

3. **모바일 디바이스에서 접근:**
   - 모바일 디바이스와 PC가 같은 WiFi 네트워크에 연결되어 있어야 합니다
   - 모바일 브라우저에서 접속: `http://192.168.0.100:8501`
   - 포트 번호는 Streamlit이 실행되는 포트로 변경하세요 (기본: 8501)

4. **방화벽 설정:**
   - Windows 방화벽에서 포트 8501 허용 필요할 수 있습니다
   - 또는 테스트 중에만 방화벽을 일시적으로 비활성화

### 방법 3: ngrok 사용 (인터넷을 통한 접근)

1. **ngrok 설치 및 사용:**
   ```bash
   # ngrok 설치 (https://ngrok.com/)
   # ngrok 실행
   ngrok http 8501
   ```
   
2. **공개 URL 받기:**
   - ngrok이 생성한 URL (예: https://xxxx.ngrok.io)을 모바일에서 접속

### 모바일 최적화 확인 사항

✅ **레이아웃:**
- 입력 폼과 콘텐츠가 세로로 잘 배치되는지
- 텍스트가 가독 가능한 크기인지
- 버튼이 터치하기 쉬운 크기인지 (최소 48px)

✅ **네비게이션:**
- 메뉴 항목들이 잘 보이는지
- 메뉴가 화면 너비를 넘지 않는지

✅ **입력 필드:**
- 입력 필드가 충분한 크기인지
- 포커스 시 자동 줌이 발생하지 않는지 (iOS, font-size: 16px 설정)

✅ **카드 및 콘텐츠:**
- 패딩이 적절한지
- 텍스트가 잘려보이지 않는지
- 이미지나 차트가 화면에 맞게 조정되는지

✅ **성능:**
- 로딩 시간이 적절한지
- 애니메이션이 부드럽게 동작하는지

### 현재 적용된 모바일 최적화

- ✅ 반응형 레이아웃 (768px, 480px 브레이크포인트)
- ✅ 터치 친화적인 버튼 크기 (최소 48px)
- ✅ iOS 자동 줌 방지 (입력 필드 font-size: 16px)
- ✅ 모바일 네비게이션 최적화
- ✅ 작은 화면용 폰트 크기 조정
- ✅ 카드 패딩 조정
- ✅ 로고 및 아이콘 크기 조정
